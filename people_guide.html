So you want to join our community's members page? This guide will walk you through the steps of forking our repository, adding your details, previewing them and making a pull request.

  • A Github Account
  • About 20 minutes
  • A photo of yourself (optional)


Œ® Fork our Repository

Log into your Github Account (or sign up for one, it's free!) and head to the repository for this website: galway-it-meetup.github.io.

In the top right, you should see the Fork button which will take you to the fork form. You can leave all of the default values and click "Create Fork".

üõ†Ô∏è Configure Github Pages

Now you have your very own copy of our website's source code. If you are familiar with Github and Jekyll, you can download the code and follow the instructions in the readme.

If this is all fairly new, you can take a shortcut: In the top menu bar, click settings, then in the left hand side menu select Pages.

Under the Build and deployment section, select "Deploy from a branch". Below you will select the branch to use - select the only option main.

The page should update to show it is building your website. The first build will take a few minutes, in the meantime we can add your profile:

üë©üèª‚Äçüíª Add your profile

Return to the homepage of your repository by clicking Code in the top menu bar. You will see the folder structure of our website. Open the _people folder and open example.md. The file will look a bit strange. In the top left above the file view you should see Preview selected. Change it to Code. You should see the same as the screenshot below.

This is an example profile. It is written in markdown, but you can also use straight-up HTML in these files.

At the top of the file, between the two sets of three dashes, is a set of conifguration called front-matter

You will make your own profile page by copying this example. Use the copy icon on the top left of the file view. Then go back to the _people folder page and in the top right, click Add File and select Create new file.

In the file path at the top of the page, enter your_name.md.

Paste in the contents of the example profile. In the front-matter, make the following changes:

  • Replace the value of name: ("You") with "Your Name" (make sure to keep the quotes)
  • Replace the value of quote: with whatever text you would like shown under your image on the people page.
  • Replace the value of image: with "/images/people/your_name.png". Later on you will upload your image to this filename. Make sure to use the file extension (.png etc) with that of the file you plan on using.

Now it's time to replace the contents of the profile below with whatever you like. Take inspiration from the other profiles already on the website. I recommend including a short bio and some links like LinkedIn, Linktree or your personal website.

You can check if your markdown is valid by switching the file view from Code to Preview. You will get a chance to view it exactly how it will be on the website and make changes later.

Save the file by pressing Commit Changes. Leave the options in the popup as they are and agan click Commit changes.

üì∏ Upload your Image

In the left hand side folder view, select images then people.

Click Add file in the top right again, but this time select the upload option. Pick you profile picture, making sure it has the same name as you set in the profile. Once again save it by using Commit Changes.

üëÄ Preview your profile

To preview your changes, go back to Settings > Pages. At the top of this page, there should now be a link to view your version of the website. Click this, and using the top left menu, go to the people page.

You should check both how your profile preview looks like on this page, and then click on it to see the full profile.

If you want to make any changes, feel free to go back to the previous steps. Remember there may be a slight delay between commiting a change and it becoming visible

üì® Pull Request

To send your changes back to our website, you can submit a Pull Request. Back on the repository page, go to Pull requests. On the new page, click the New pull request button.

As you forked our website's repository, the default options should all be correct, similar to the screensot above. The base repository belongs to galway-it while the head is your fork.

Below you can see all of the changes you have made. If you are happy with them, click Create pull request. You will have to add a title - just use your name here. Then click Create pull request again.

üéâ Success

Congrats! You have made a code contribution to an Open Source Repository! Now we will get an email notification and have to review and approve your changes. If there's any questions, we can reply through the Pull Request.

Within a week or so your changes should be merged and live. If not, feel free to reach out to us via our socials.
